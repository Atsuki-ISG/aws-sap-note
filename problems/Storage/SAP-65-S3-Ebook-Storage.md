# Problem Statement

【SAP-65】 あるIT系出版社は世界各国の編集者と読者に電子書籍を配信するSaaSをAWSへ再構築している。
ユーザーはWebポータルから原稿や画像をアップロードし、校正後にPDFをダウンロードする。
本番環境は運用要員3名で24時間体制を維持しつつ、繁忙期にストレージ需要が瞬時に倍増しても自動拡張できることが求められている。
経営層は「インフラ保守や暗号化機能の自社実装に人的コストを割けない」と判断し、移行フェーズ・運用フェーズともフルマネージドサービスのみを利用するとのポリシーを策定した。
さらに著作権侵害リスクを抑止するため、保存データも転送データも常時暗号化し、暗号鍵は社内セキュリティ部門が所有して年次ローテーションすることを必須要件としている。
電子書籍原稿は失われると損害が甚大であるため、99.999999999%の耐久性と高可用性という技術的条件も外せない。
これらのビジネス背景と技術制約を満たすため、ソリューションアーキテクトはインフラ設計レベルでどのAWSサービスと暗号化方式を採用すべきか。
最適な選択肢はどれか？

# Options

A. EC2 インスタンスに接続された EBS ボリュームに原稿を保存し、EBS 暗号化に KMS キーを用いる。暗号化通信は EC2 上のアプリケーションで独自に実装する。
B. Storage Gateway ファイルゲートウェイを使って NFS 経由でデータを取り込み、バックエンドの EBS ボリュームを AWS KMS キーで暗号化する。
C. Amazon S3 に電子書籍ファイルを保存し、バケットポリシーで HTTPS 接続を必須とする。保存時は AWS KMS カスタマー管理キーによる SSE-KMS をデフォルト暗号化として強制する。
D. 原稿データを Amazon DynamoDB に保存し、SSL 経由のアクセスと KMS による暗号化（DynamoDB の暗号化機能）を有効化する。

# Answer
C
